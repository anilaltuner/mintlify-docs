## Webhook Events

The integration processes real-time events from Booxi webhooks:

<AccordionGroup>
  <Accordion title="appointment.created" icon="calendar-plus">
    **Triggered:** When a new booking is created
    
    **Actions:**
    - Store complete appointment details
    - Link to customer profile
    - Enable future booking retrieval
    - Send confirmation notification
    
    **Data Received:**
    - Appointment ID and confirmation number
    - Service details (name, duration, price)
    - Staff assignment
    - Location and time
    - Customer information
  </Accordion>

  <Accordion title="appointment.remind" icon="bell">
    **Triggered:** Based on reminder schedule (default: 30 minutes before)
    
    **Actions:**
    - Send customized reminder message
    - Include all booking details
    - Support custom message templates
    
    **Use Case:**
    Reduce no-shows by sending timely reminders with location and preparation details.
  </Accordion>

  <Accordion title="appointment.status_updated" icon="arrows-rotate">
    **Triggered:** When appointment status changes
    
    **Statuses:**
    - `Pending` ‚Üí `Confirmed`
    - `Confirmed` ‚Üí `Approved`
    - `Approved` ‚Üí `Cancelled`
    
    **Actions:**
    - Notify customer of status change
    - Update internal booking records
    - Execute status-based workflows
  </Accordion>

  <Accordion title="appointment.updated" icon="pen-to-square">
    **Triggered:** When booking details are modified
    
    **Actions:**
    - Detect rescheduling and send notifications
    - Update service or staff assignments
    - Maintain booking history
    
    **Common Updates:**
    - Time changes (rescheduling)
    - Staff reassignment
    - Service modifications
    - Location changes
  </Accordion>
</AccordionGroup>

## Configuration

### Prerequisites

<Steps>
  <Step title="Booxi Account">
    Active Booxi account with API access enabled
  </Step>
  <Step title="Configure Services">
    Services, staff, and locations set up in Booxi dashboard
  </Step>
  <Step title="API Credentials">
    Obtain API Key and Partner Key for each market
  </Step>
  <Step title="Webhook Access">
    Webhook endpoint permissions configured
  </Step>
</Steps>

### Required Credentials

<Info>
Credentials are required **per market/location**. Multi-market setups need separate keys for each region.
</Info>

<CodeGroup>
```json Configuration Format
{
  "api_key": "your-booxi-api-key",
  "partner_key": "your-booxi-partner-key",
}
```

```bash Environment Variables
BOOXI_UK_API_KEY=your-uk-api-key
BOOXI_UK_PARTNER_KEY=your-uk-partner-key
BOOXI_DE_API_KEY=your-de-api-key
BOOXI_DE_PARTNER_KEY=your-de-partner-key
```
</CodeGroup>

### Setup Steps

<Steps>
  <Step title="Provide Credentials">
    Submit API keys for each market/region you operate in
  </Step>
  <Step title="Knowledge Base Update">
    System syncs your service catalog and stores from Booxi automatically
  </Step>
  <Step title="Customize Messages (Optional)">
    Configure custom notification templates using dynamic variables
  </Step>
  <Step title="Webhook Configuration">
    Configure webhook endpoints and event subscriptions
  </Step>
  <Step title="Testing">
    Verify end-to-end flows in test environment
  </Step>
  <Step title="Go Live">
    Deploy to production channels
  </Step>
</Steps>
